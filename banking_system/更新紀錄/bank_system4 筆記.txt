User 類別是用來存儲使用者的 ID 和密碼的。

DatabaseHandler 類別被用來與數據庫進行互動。它首先確認使用者 ID 和密碼，然後讀取使用者的餘額。如果使用者試圖取款，它將檢查餘額是否足夠。如果足夠，則它將更新數據庫中的餘額。

BankingSystem 類別控制程序的流程。在主方法中，它首先提示使用者輸入 ID 和密碼，然後嘗試取款，最後如果取款成功，則更新餘額。

功能地圖:
- 登入功能:
  - 驗證使用者身份
  - 檢查帳號密碼是否正確

- 提款功能:
  - 輸入提款金額
  - 檢查提款金額是否超過帳戶餘額
  - 更新帳戶餘額
  - 新增提款交易記錄

- 存款功能:
  - 輸入存款金額
  - 更新帳戶餘額
  - 新增存款交易記錄

- 轉帳功能:
  - 輸入轉帳金額和收款帳號
  - 檢查轉帳金額是否超過帳戶餘額
  - 更新帳戶餘額
  - 新增轉帳交易記錄

- 查詢餘額功能:
  - 顯示帳戶餘額

- 退出功能:
  - 關閉程式

類別:
- User:
  - 屬性: 使用者ID、密碼、卡號
  - 方法: 驗證密碼、設置密碼

- DatabaseHandler:
  - 方法: 執行查詢、執行更新、關閉資源

- HashUtil:
  - 方法: 生成鹽值、計算哈希密碼

- JFrame_1:
  - 屬性: GUI元件
  - 方法: 監聽按鈕事件、插入交易紀錄
